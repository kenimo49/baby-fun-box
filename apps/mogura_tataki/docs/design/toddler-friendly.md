# 幼児向け設計思想

## 概要

もぐらたたきは、1〜2歳児が楽しく遊べるよう設計されています。このドキュメントでは、その設計判断の背景を説明します。

## プレッシャーのない設計

### スコアなし

**なぜスコアがないのか？**

- 1〜2歳児は数字や点数の概念をまだ理解していない
- 競争や評価のプレッシャーは幼児にストレスを与える
- 「遊び」自体が目的であり、結果は重要でない

### タイマーなし

**なぜタイマーがないのか？**

- 時間制限は焦りを生む
- 幼児は自分のペースで探索することで学ぶ
- いつでも中断・再開できる安心感

### 失敗がない

**なぜ失敗がないのか？**

- タッチしなくても動物は自然に引っ込む
- 「間に合わなかった」という概念がない
- すべての操作が「成功」として扱われる

## ゆっくりしたペース

### 表示時間

```
MIN_SHOW_TIME = 2.0秒
MAX_SHOW_TIME = 4.0秒
```

**設計理由**:

1. 幼児の反応速度は大人より遅い
2. 見つけて → 認識して → 手を動かす に時間がかかる
3. 2〜4秒あれば、ほとんどの幼児がタッチできる

### 出現間隔

```
SPAWN_INTERVAL = 1.5秒
```

**設計理由**:

1. 次の動物を「待つ」時間が楽しい
2. 「どこから出るかな？」というワクワク感
3. 情報過多を防ぐ

### 最大同時表示

```
MAX_ACTIVE = 3体
```

**設計理由**:

1. 多すぎると注意が分散する
2. 幼児は1つに集中するのが得意
3. 画面がごちゃごちゃしない

## 視覚デザイン

### 大きなキャラクター

```
HOLE_SIZE = 140px
CHAR_SIZE = 100px
```

**設計理由**:

1. 幼児の視力はまだ発達途中
2. 大きいほうが認識しやすい
3. タッチ範囲も大きくなる

### カラフルな動物

| キャラクター | 色 | 理由 |
|-------------|-----|------|
| うさぎ | ピンク | 親しみやすい |
| くま | オレンジ | 暖かみがある |
| ねこ | きいろ | 明るく目立つ |
| いぬ | みずいろ | 落ち着きと爽やかさ |
| ひよこ | きいろ | かわいらしさ |
| かえる | みどり | 自然のイメージ |

**色の設計理由**:

1. BABY_COLORS パレットを使用
2. 高コントラストで見やすい
3. 各キャラクターが区別しやすい

### シンプルな背景

- 薄いグレーの背景
- 茶色の穴（土をイメージ）
- 余計な装飾なし

**設計理由**:

1. キャラクターが目立つ
2. 注意が分散しない
3. 視覚的ノイズを最小化

## 聴覚フィードバック

### 出現音（ポップ音）

- 高めの周波数（600Hz）
- 短い持続時間（0.1秒）
- 明るく軽い音

**設計理由**:

1. 「何か出てきた！」という注目を引く
2. 怖がらせない優しい音
3. 聴覚的な手がかり

### タップ音（達成音）

- 上昇する周波数（600→900Hz）
- やや長め（0.15秒）
- 達成感のある音

**設計理由**:

1. 「できた！」という成功体験
2. ポジティブな強化
3. 操作と結果の因果関係を学ぶ

## タッチ操作の設計

### 大きな判定領域

```python
rect = pygame.Rect(
    x - CHAR_SIZE // 2,
    y - CHAR_SIZE // 2,
    CHAR_SIZE,
    CHAR_SIZE
)
```

**設計理由**:

1. 幼児の指は小さいが、精度は低い
2. 大きな範囲でタッチを受け付ける
3. 「外した」という失敗感を減らす

### 即座のフィードバック

- タッチ → 即座にバウンスアニメーション
- タッチ → 即座に音が鳴る
- タッチ → 即座にパーティクル

**設計理由**:

1. 操作と結果の関連を明確に
2. 「自分がやった」という実感
3. 遅延はフラストレーションの原因

## 視覚的フィードバック

### バウンスアニメーション

```python
bounce = math.sin(tap_animation * math.pi) * 30
```

- タップすると上に跳ねる
- 楽しい動き
- 「やった！」という視覚的報酬

### パーティクル

- カラフルな粒子が飛び散る
- 8個の粒子
- 0.5秒で消える

**設計理由**:

1. お祝い感を演出
2. 視覚的な報酬
3. 短時間で消えて次に集中できる

## 親しみやすいキャラクター

### 動物の選定基準

1. **馴染みのある動物**: 絵本や日常で見る動物
2. **怖くない**: 猛獣や虫は避ける
3. **かわいい**: 丸みのある形状

### 表情の設計

- 大きな目（かわいらしさ）
- 丸い顔（親しみやすさ）
- シンプルな形（認識しやすさ）

## まとめ

| 設計要素 | 実装 | 目的 |
|----------|------|------|
| スコア | なし | プレッシャー軽減 |
| タイマー | なし | 自由な探索 |
| 失敗 | なし | 自己肯定感 |
| 表示時間 | 2〜4秒 | 余裕を持って遊べる |
| キャラサイズ | 100px | 認識しやすい |
| 判定範囲 | 140px | タッチしやすい |
| フィードバック | 即座 | 因果関係の学習 |
| 音 | 優しい音 | ポジティブな体験 |

すべての設計判断は「幼児が楽しく、ストレスなく遊べる」という目標に基づいています。
