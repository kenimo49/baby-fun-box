# 幼児向け設計思想

## 概要

もぐらたたきは、1〜2歳児が楽しく遊べるよう設計されています。このドキュメントでは、その設計判断の背景を説明します。

## 30秒タイマーの設計

### なぜ30秒なのか？

**赤ちゃんの集中力に合わせた設計**:

1. 1〜2歳児の集中力は非常に短い（数十秒〜数分）
2. 60秒では長すぎて飽きてしまう可能性
3. 30秒なら最後まで集中して遊べる
4. 短いサイクルで「達成感」を何度も味わえる

### タイマー表示の工夫

- **大きな数字**: 見やすいフォントサイズ
- **残り10秒で赤色**: 終わりが近いことを視覚的に伝える
- **シンプルな秒表示**: 複雑な表示は避ける

## スコアシステムの設計

### なぜスコアを導入したのか？

**ポジティブな体験のため**:

1. タッチの成功を数値で確認できる
2. 「たくさんタッチできた！」という達成感
3. 親子で一緒に数を確認できる

### 褒め言葉の設計

| スコア | メッセージ | 設計意図 |
|--------|----------|---------|
| 15点以上 | すごーい！ | 高い達成感 |
| 10〜14点 | がんばったね！ | 努力を認める |
| 5〜9点 | いいね！ | ポジティブな評価 |
| 0〜4点 | たのしかったね！ | 結果より過程を重視 |

**重要な設計方針**:

- **失敗を示す言葉は使わない**
- **どんなスコアでもポジティブな言葉**
- **「たのしかったね！」が最低評価**（遊んだこと自体を肯定）

## ゲームフローの設計

### 3画面構成

```
スタート画面 → プレイ画面 → 結果画面 → スタート画面...
```

**なぜこの構成か？**

1. **明確な開始と終了**: ゲームの区切りが分かりやすい
2. **リプレイしやすい**: 「もういちど」ボタンで簡単に再開
3. **達成感の演出**: 結果画面で褒められる体験

### スタート画面の設計

- **シンプルな説明**: 「30びょうで どうぶつを たくさんタッチしよう！」
- **大きなスタートボタン**: タッチしやすい
- **穴は表示しない**: ゲーム開始前の期待感を高める

### 結果画面の設計

- **大きなスコア表示**: 達成感を視覚的に
- **褒め言葉**: ポジティブなフィードバック
- **ファンファーレ音**: 達成感を聴覚的にも

## ゆっくりしたペース

### 表示時間

```
MIN_SHOW_TIME = 2.0秒
MAX_SHOW_TIME = 4.0秒
```

**設計理由**:

1. 幼児の反応速度は大人より遅い
2. 見つけて → 認識して → 手を動かす に時間がかかる
3. 2〜4秒あれば、ほとんどの幼児がタッチできる

### 出現間隔

```
SPAWN_INTERVAL = 1.5秒
```

**設計理由**:

1. 次の動物を「待つ」時間が楽しい
2. 「どこから出るかな？」というワクワク感
3. 情報過多を防ぐ

### 最大同時表示

```
MAX_ACTIVE = 3体
```

**設計理由**:

1. 多すぎると注意が分散する
2. 幼児は1つに集中するのが得意
3. 画面がごちゃごちゃしない

## 視覚デザイン

### 大きなキャラクター

```
HOLE_SIZE = 140px
CHAR_SIZE = 100px
```

**設計理由**:

1. 幼児の視力はまだ発達途中
2. 大きいほうが認識しやすい
3. タッチ範囲も大きくなる

### カラフルな動物

| キャラクター | 色 | 理由 |
|-------------|-----|------|
| うさぎ | ピンク | 親しみやすい |
| くま | オレンジ | 暖かみがある |
| ねこ | きいろ | 明るく目立つ |
| いぬ | みずいろ | 落ち着きと爽やかさ |
| ひよこ | きいろ | かわいらしさ |
| かえる | みどり | 自然のイメージ |

**色の設計理由**:

1. BABY_COLORS パレットを使用
2. 高コントラストで見やすい
3. 各キャラクターが区別しやすい

### シンプルな背景

- 草原をイメージした緑のグラデーション
- 茶色の穴（土をイメージ）
- 余計な装飾なし

**設計理由**:

1. キャラクターが目立つ
2. 注意が分散しない
3. 視覚的ノイズを最小化

## 聴覚フィードバック

### 出現音（ポップ音）

- 高めの周波数（400→800Hz 上昇）
- 短い持続時間（0.1秒）
- 明るく軽い音

**設計理由**:

1. 「何か出てきた！」という注目を引く
2. 怖がらせない優しい音
3. 聴覚的な手がかり

### タップ音（達成音）

- 明るい和音（C, E, G）
- やや長め（0.15秒）
- 達成感のある音

**設計理由**:

1. 「できた！」という成功体験
2. ポジティブな強化
3. 操作と結果の因果関係を学ぶ

### 終了音（ファンファーレ）

- 上昇するメロディ（G→C→E→G）
- 長め（0.8秒）
- お祝い感のある音

**設計理由**:

1. ゲーム終了を明確に伝える
2. 達成感を演出
3. ポジティブな締めくくり

## タッチ操作の設計

### 大きな判定領域

```python
face_rect = pygame.Rect(
    hole.x - 50,
    hole.y - 80,
    100,
    100
)
```

**設計理由**:

1. 幼児の指は小さいが、精度は低い
2. 大きな範囲でタッチを受け付ける
3. 「外した」という失敗感を減らす

### 即座のフィードバック

- タッチ → 即座にバウンスアニメーション
- タッチ → 即座に音が鳴る
- タッチ → 即座にパーティクル
- タッチ → 即座にスコア+1

**設計理由**:

1. 操作と結果の関連を明確に
2. 「自分がやった」という実感
3. 遅延はフラストレーションの原因

## 視覚的フィードバック

### バウンスアニメーション

```python
bounce = math.sin(tap_animation * math.pi) * 30
```

- タップすると上に跳ねる
- 楽しい動き
- 「やった！」という視覚的報酬

### パーティクル

- カラフルな粒子が飛び散る
- 8個の粒子
- 0.5秒で消える

**設計理由**:

1. お祝い感を演出
2. 視覚的な報酬
3. 短時間で消えて次に集中できる

## 親しみやすいキャラクター

### 動物の選定基準

1. **馴染みのある動物**: 絵本や日常で見る動物
2. **怖くない**: 猛獣や虫は避ける
3. **かわいい**: 丸みのある形状

### 表情の設計

- 大きな目（かわいらしさ）
- 丸い顔（親しみやすさ）
- シンプルな形（認識しやすさ）

## まとめ

| 設計要素 | 実装 | 目的 |
|----------|------|------|
| ゲーム時間 | 30秒 | 集中力に合わせる |
| スコア | あり | 達成感の可視化 |
| 褒め言葉 | 全スコアでポジティブ | 自己肯定感 |
| 表示時間 | 2〜4秒 | 余裕を持って遊べる |
| キャラサイズ | 100px | 認識しやすい |
| 判定範囲 | 100x100px | タッチしやすい |
| フィードバック | 即座 | 因果関係の学習 |
| 音 | 優しい音 | ポジティブな体験 |

すべての設計判断は「幼児が楽しく、達成感を感じながら遊べる」という目標に基づいています。
